PROFILING TOOLS REFERENCE
=========================

Each entry: tool name, one-liner setup/invocation, output format.

BROWSER
-------

Chrome DevTools Performance
  One-liner: Open DevTools (F12) -> Performance tab -> Record (circle icon) ->
             interact with page -> Stop.
  Output:   Flame chart with Main thread, Network, Timings. Bottom-up /
            Call Tree / Event Log tabs for analysis.

Lighthouse CLI
  One-liner: npx lighthouse https://example.com --view --preset=desktop
  Output:   HTML report with 0-100 scores for Performance, Accessibility,
            Best Practices, SEO, PWA. Recommendations with estimated savings.

WebPageTest
  One-liner: curl "https://www.webpagetest.org/runtest.php?url=https://example.com&f=json"
             (then poll for results via the returned JSON URL)
  Output:   JSON with Timings, Requests breakdown, Filmstrip, Performance
            Optimization checks.

Lighthouse CI
  One-liner: npx lhci autorun --config=lighthouserc.json
  Output:   JSON summary with assertion results (pass/warn/error) per metric.
            Uploads to LHCI server if configured.

NODE.JS
-------

clinic.js doctor
  One-liner: npx clinic doctor -- node app.js
  Output:   HTML report with flame graph, event loop delay chart, memory usage
            over time, and doctor recommendations.

clinic.js flame
  One-liner: npx clinic flame -- node app.js
  Output:   Interactive flame graph (D3.js) showing stack samples over time.
            Helps identify hot functions.

clinic.js bubbleprof
  One-liner: npx clinic bubbleprof -- node app.js
  Output:   Bubble profile HTML showing async operation boundaries and
            latency between operations.

0x (flame graph profiler)
  One-liner: npx 0x node app.js
  Output:   Interactive flame graph (SVG/HTML). Stacks collected via
            perf (Linux) or DTrace (macOS).

Node.js --prof
  One-liner: node --prof app.js && node --prof-process isolate*.log > processed.txt
  Output:   Text summary of V8 tick samples by JS function, C++ function,
            and library.

PYTHON
------

cProfile
  One-liner: python -m cProfile -o profile.out script.py
             python -m pstats profile.out
  Output:   ncalls, tottime, percall, cumtime, percall, filename:lineno(function).
            Sortable by time, calls, cumulative time.

py-spy
  One-liner: sudo py-spy record -o profile.svg --pid <PID>
             py-spy top --pid <PID>
  Output:   Flame graph SVG (record mode) or live top-like display (top mode).
            Sampled profiler - no code modification needed.

Yappi
  One-liner: python -m yappi -f callgrind -o profile.out script.py
  Output:   Callgrind format - view with qcachegrind or kcachegrind.
            Wall-clock or CPU profiling.

GO
--

⚠️  PRODUCTION WARNING: The /debug/pprof/ endpoints expose stack traces, heap
    contents, goroutine listings, and potentially sensitive in-memory data.
    Never register pprof handlers on public interfaces. Bind only to 127.0.0.1
    and restrict via firewall or authentication. Use build tags to exclude
    profiling code from production builds.

pprof (built-in net/http/pprof)
  One-liner: go tool pprof -http=:8080 http://localhost:6060/debug/pprof/profile?seconds=30
  Output:   Web UI with flame graph, top, graph, peek, source views.
            Can also generate SVG, PDF, or text.

pprof (heap)
  One-liner: go tool pprof -http=:8080 http://localhost:6060/debug/pprof/heap
  Output:   Memory allocation flame graph, inuse/alloc space, alloc count.

Go execution tracer
  One-liner: curl http://localhost:6060/debug/pprof/trace?seconds=5 > trace.out
             go tool trace trace.out
  Output:   Web UI with goroutine analysis, blocking profile, network
            wait profile, syscall profile, scheduler latency.

pprof (mutex)
  One-liner: go tool pprof -http=:8080 http://localhost:6060/debug/pprof/mutex
  Output:   Stack traces with mutex contention count and duration.

UNIVERSAL
---------

⚠️  PRODUCTION WARNING: sudo perf/strace/bpftrace have full kernel and process
    visibility (including encryption keys, tokens, and file contents in memory).
    Never run on production unless as a last resort with explicit security
    approval. Ensure no sensitive data is in process memory during profiling.

perf (Linux)
  One-liner: sudo perf record -F 99 -p <PID> -g -- sleep 30
             sudo perf report -n --stdio
             # or generate flamegraph:
             sudo perf script | stackcollapse-perf.pl | flamegraph.pl > flame.svg
  Output:   Text report with call chains and sample counts. Can generate
            interactive flame graphs with FlameGraph toolkit.

strace
  One-liner: strace -c -p <PID>   # summary of syscalls
             strace -T -f -p <PID> # with timestamps per syscall
  Output:   System call counts, errors, and time. Good for identifying
            excessive I/O syscalls or file operations.

bpftrace (eBPF)
  One-liner: bpftrace -e 'kprobe:do_sys_open { printf("open: %s\n", str(arg1)); }'
  Output:   Dynamic trace output. Can probe kernel and user-space functions
            with minimal overhead.

heaptrack (Linux memory)
  One-liner: heaptrack ./myapp
  Output:   Heap profiler output (GUI with heaptrack_gui) showing allocation
            hotspots, temporary allocations, and leak candidates.
