# Spec Document Review

## Context
Review the following spec document for quality and completeness before implementation begins.

## Spec to Review
{SPEC_CONTENT}

## Review Criteria

### Completeness
- Are all requirements explicitly stated?
- Are there any "TBD", "TODO", or placeholder sections?
- Are edge cases and error states addressed?

### Consistency
- Do any sections contradict each other?
- Does the architecture match the feature descriptions?
- Are terms used consistently throughout?

### Clarity
- Could any requirement be interpreted two different ways?
- Are technical terms defined?
- Is the scope clearly bounded?

### Feasibility
- Is the proposed approach technically sound?
- Are dependencies and constraints identified?
- Are there any hidden complexities?

## Output
Return: Assessment with issues by category, severity (Critical/Important/Minor), and overall readiness rating.
